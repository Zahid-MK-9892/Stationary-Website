# MERN Freelancing Studio Website - Complete Setup Guide

## Prerequisites
Before you begin, ensure you have the following installed:
- Node.js (v16 or higher) - https://nodejs.org/
- MongoDB (v5 or higher) - https://www.mongodb.com/try/download/community
- Git - https://git-scm.com/
- A code editor (VS Code recommended) - https://code.visualstudio.com/

## Project Structure Overview
```
/workspace
├── client/          # React frontend application
├── server/          # Node.js/Express backend API
├── database/        # MongoDB connection and models
└── README.md        # This file
```

## Step-by-Step Execution Process

### Phase 1: Environment Setup

#### 1. Install Dependencies for Backend
Open your terminal in the `/workspace` directory and run:

```bash
cd server
npm install
```

This will install all necessary backend packages including:
- Express.js (web framework)
- Mongoose (MongoDB ODM)
- CORS (cross-origin resource sharing)
- Dotenv (environment variables)
- Nodemon (development auto-restart)

#### 2. Install Dependencies for Frontend
Open a new terminal window (keep the first one open) and run:

```bash
cd client
npm install
```

This will install all necessary frontend packages including:
- React
- React Router DOM
- Axios (HTTP client)
- And other React dependencies

### Phase 2: Database Configuration

#### Option A: Using Local MongoDB
1. Start your MongoDB service:
   - **Windows**: `net start MongoDB`
   - **Mac**: `brew services start mongodb-community`
   - **Linux**: `sudo systemctl start mongod`

2. Verify MongoDB is running:
   ```bash
   mongosh
   ```
   You should see the MongoDB shell prompt. Type `exit` to leave.

#### Option B: Using MongoDB Atlas (Cloud)
1. Go to https://www.mongodb.com/cloud/atlas
2. Create a free account
3. Create a new cluster
4. Get your connection string
5. Update the `.env` file in the server folder with your connection string

### Phase 3: Environment Variables Configuration

#### 1. Configure Backend Environment
Create a file named `.env` in the `/workspace/server` directory with the following content:

```
PORT=5000
MONGODB_URI=mongodb://localhost:27017/freelance-studio
NODE_ENV=development
```

If using MongoDB Atlas, replace the MONGODB_URI with your connection string:
```
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/freelance-studio?retryWrites=true&w=majority
```

#### 2. Configure Frontend Environment (Optional)
Create a file named `.env` in the `/workspace/client` directory with:

```
REACT_APP_API_URL=http://localhost:5000/api
```

### Phase 4: Running the Application

You need to run both the backend and frontend simultaneously. Here are two methods:

#### Method A: Running in Two Separate Terminals (Recommended for Development)

**Terminal 1 - Backend Server:**
```bash
cd /workspace/server
npm run dev
```
You should see: "Server running on port 5000" and "MongoDB connected successfully"

**Terminal 2 - Frontend Client:**
```bash
cd /workspace/client
npm start
```
Your browser should automatically open to `http://localhost:3000`

#### Method B: Running with Concurrent (Alternative)

Install concurrent globally if not already installed:
```bash
npm install -g concurrently
```

Then from the root `/workspace` directory:
```bash
concurrently "npm run server" "npm run client"
```

*Note: This requires package.json scripts to be set up in the root directory*

### Phase 5: Testing the Application

1. **Homepage**: Navigate to `http://localhost:3000`
   - Verify the hero section loads
   - Check navigation menu
   - Scroll through featured work section

2. **Portfolio Page**: Click "Work" or navigate to `/work`
   - Verify project cards display
   - Test filtering functionality (if implemented)

3. **Contact/Project Form**: Click "Hire Me" or navigate to `/contact`
   - Fill out the form with test data
   - Submit the form
   - Check backend terminal for successful POST request
   - Verify data is saved in MongoDB

4. **API Testing** (Optional):
   Use Postman or curl to test endpoints:
   ```bash
   # Test getting all projects
   curl http://localhost:5000/api/projects
   
   # Test submitting a project request
   curl -X POST http://localhost:5000/api/contact \
     -H "Content-Type: application/json" \
     -d '{"name":"Test User","email":"test@example.com","message":"Test message"}'
   ```

### Phase 6: Building for Production

When ready to deploy:

#### 1. Build Frontend
```bash
cd /workspace/client
npm run build
```
This creates an optimized production build in the `client/build` folder.

#### 2. Configure Backend to Serve Static Files
Ensure your server.js serves the React build files in production mode.

#### 3. Start Production Server
```bash
cd /workspace/server
npm start
```

### Common Issues & Solutions

#### Issue 1: MongoDB Connection Error
**Error**: "MongooseError: connect ECONNREFUSED"
**Solution**: 
- Ensure MongoDB service is running
- Check MONGODB_URI in .env file
- Verify port 27017 is not blocked by firewall

#### Issue 2: Port Already in Use
**Error**: "Port 5000 is already in use" or "Port 3000 is already in use"
**Solution**:
```bash
# Find process using the port
lsof -i :5000  # or :3000

# Kill the process
kill -9 <PID>
```
Or change the port in .env file

#### Issue 3: CORS Errors
**Error**: "Access-Control-Allow-Origin" errors in browser console
**Solution**:
- Ensure cors middleware is properly configured in server.js
- Verify frontend API calls point to correct backend URL

#### Issue 4: Module Not Found
**Error**: "Cannot find module"
**Solution**:
- Delete node_modules folder
- Run `npm install` again
- Clear npm cache: `npm cache clean --force`

### Deployment Options

#### Option 1: Heroku
1. Install Heroku CLI
2. Create Procfile in root directory
3. Deploy using git push heroku main

#### Option 2: Vercel (Frontend) + Railway/Render (Backend)
1. Push code to GitHub
2. Connect Vercel to client folder
3. Connect Railway/Render to server folder
4. Set environment variables in respective platforms

#### Option 3: DigitalOcean/VPS
1. Set up Ubuntu server
2. Install Node.js, MongoDB, Nginx
3. Clone repository
4. Configure PM2 for process management
5. Set up Nginx as reverse proxy

### Security Best Practices

1. Never commit .env files to Git
2. Use environment variables for sensitive data
3. Implement rate limiting on API endpoints
4. Add input validation and sanitization
5. Use HTTPS in production
6. Regularly update dependencies
